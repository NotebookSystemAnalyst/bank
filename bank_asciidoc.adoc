== Алгоритм работы метода
. Метод получает на вход токен аутентификации.
. Проверяет его валидность.
** 2.1 Если токен не валиден, возвращается ответ 403.
** 2.2 Если токен валиден, то происходит проверка на наличие обязательных атрибутов.
. Если хотя бы один обязательный атрибут не найден, возвращается ответ 400.
. Если все обязательные атрибуты указаны, то происходит создание нового ресурса с параметрами, переданными в теле запроса.

== Возможные ошибки
|===

| *Код ошибки* | *Текст ошибки* | *Причина*

|400
|bad request
|Не переданы обязательные атрибуты запроса

|400
|unauthorized
|Пользователь не авторизован или у токена истек срок действия

|===

== Пример запроса

[source,bash]
----
curl --location --request POST /ru-external-api/client-fast-check' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer sadfdsafzxcvq21321dsa' \
--data-raw '
{
    "testString": "some test",
    "testBoolean": true,
    "testNumber": 1234,
    "testArray": [],
    "testObject": {}
}
----
